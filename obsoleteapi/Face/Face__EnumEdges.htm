Face::EnumEdges

This method is obsolete and has been superseded by Face2::EnumEdges.

Description

This method enumerates the edges in a face.

 

Syntax (OLE Automation)

Not available.

 

 

Syntax (COM)

status = Face->EnumEdges ( &retval )

 

Output:

(LPENUMEDGES) retval

Pointer to the enumerated list of edges

Return:

(HRESULT) status

S_OK if successful

 

Remarks

If you try to align the edges with their corresponding trim curves, then call EnumEdges immediately following a call to GetTrimCurves. This causes the order of the trim curves to align with the order of the edges. For Dispatch implementations, refer to GetEdges.

To elaborate, any call to GetTrimCurves stores a temporary buffer that contains the edges for this face. If this temporary buffer exists, then GetEdges or EnumEdges access the buffer. This is beneficial if you try to align the trim curves of the face with their corresponding edges. If you call GetEdges or EnumEdges immediately following GetTrimCurves, then the order of the trim curves aligns with the order of the edges. However, if you process all the trimcurves and then process all the edges, your first call to EnumEdges uses the temporary buffer of edges generated in your last call to GetTrimCurves. Since your last call to GetTrimCurves and your current call to EnumEdges are probably made from different faces, you will see a mismatch of data. This mismatch of data may been seen when GetEdgeCount does not match the number of enumerated edges or it may be seen when the edge data does not match the face being examined. Therefore, calls to GetEdges or EnumEdges should be made in the same loop and immediately following your call to GetTrimCurves. The temporary buffer of edges is destroyed after each call to GetEdges or EnumEdges.

If you are using GetTrimCurves, also be aware that the number of trim curves on a face may vary from the number of edges on the face. If you call EnumEdges immediately after a call to GetTrimCurves, then you will be getting a list of edges generated by GetTrimCurves. Because this list of generated edges may vary from the list of edges that would typically be returned from this face, you should not use Face::GetEdgeCount to determine the length of the enumerated edge list. Instead, you can determine the size of the enumerated edge list by using one of the return values from GetTrimCurves. The GetTrimCurves method returns the number of trim (SP) curves in the first array position. This is the value which should be used to loop through your enumerated list of edges.

For more information on aligning the edges with the trim curves, refer to the documentation of GetTrimCurves.
