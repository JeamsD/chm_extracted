Face::GetEdges

This method is obsolete and has been superseded by Face2::GetEdges.

Description

This method returns an array of pointers to the edges bounding this face.

 

Syntax (OLE Automation)

retval = Face.GetEdges ()

 

Return:

(VARIANT) retval

VARIANT of type SafeArray of an array of pointers to Dispatch objects

 

Syntax (COM)

status = Face->IGetEdges ( edgeList )

 

Output:

(LPEDGE*) edgeList

Pointer to an array of edges

Return:

(HRESULT) status

S_OK if successful

 

Remarks

If you try to align the edges with their corresponding trim curves, then do not call GetEdges immediately following a call to GetTrimCurves. This causes the order of the trim curves to align with the order of the edges. The COM implementation of this method, IGetEdges, does not align the returned edges with their corresponding trim curves. If you are using COM and you need the edges aligned with their trim curves, use EnumEdges.

To elaborate, any call to GetTrimCurves stores a temporary buffer containing the edges for this face. If this temporary buffer exists, then calls to GetEdges or EnumEdges access the buffer. This is beneficial if you want to align the trim curves of the face with their corresponding edges. If you call GetEdges or EnumEdges immediately following a call to GetTrimCurves, then the order of the trim curves aligns with the order of the edges. However, if you are processing all the trimcurves and then processing all the edges, your first call to GetEdges uses the temporary buffer of edges generated in your last call to GetTrimCurves. Since your last call to GetTrimCurves and your current call to GetEdges are probably made from different face objects, these method return mismatched data. This mismatch occurs when the value returned by GetEdgeCount does not match the SafeArray size from GetEdges, or when the edge data does not match the examined face. Therefore, call GetEdges or EnumEdges in the same loop and immediately following a call to GetTrimCurves. The temporary buffer of edges is destroyed after each call to GetEdges or EnumEdges.

If you are using GetTrimCurves, also be aware that the number of trim curves on a face can vary from the number of edges on the face. If you call GetEdges immediately after calling GetTrimCurves, then you are getting the list of edges generated by GetTrimCurves. Because this list of generated edges can vary from the list of edges that is typically returned from this face, do not use GetEdgeCount to determine the length of this edge list. Instead, you can determine the size of the edge list using one of the return values from GetTrimCurves or by getting the upper bound of the GetEdges SafeArray. (GetTrimCurves returns the number of trim (SP) curves in the first array position. Use this value to loop through your list of edges.)

For more information on aligning the edges with the trim curves, see GetTrimCurves.
